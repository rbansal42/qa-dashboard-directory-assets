Drillthrough binding 02

Drillthrough is what makes a single page render dynamically for any selected dashboard. Configure it before placing visuals — every visual you add afterward will inherit the filter.

2.1 Bind DashboardName as the drillthrough field

  1. With the Dashboard Detail page active and no visual selected, open Visualizations → Drill through
  2. Set Cross-report = Off
  3. Set Keep all filters = Off
  4. Drag DashboardDirectory[DashboardName] into the Add drill-through fields here well
What this unlocks Every visual on this page is now implicitly filtered to the single row whose DashboardName the user clicked from the landing-page matrix. No card needs an explicit filter; the page filter does the work.

2.2 Auto-generated back button

Power BI inserts a back button at X=14, Y=14 automatically. Move it later (Step 05) — keep the page-back behaviour.

2.3 Wire the matrix on Page 1 to call this page

Switch back to the landing page. Drillthrough is invoked by right-click on a row in the dashboard matrix. To make it discoverable:

  1. Add a small "Right-click → Drill through → Dashboard Detail" hint as a Card visual or text box near the matrix header
  2. Optionally insert a small drillthrough icon button per row using a calculation group; keep right-click as the canonical path
Critical Leave Keep all filters Off. If you turn it On, the search slicer text and category pill from the landing page leak in and corrupt the meta bar.

3.4 Detail-page measures · paste into DashboardDirectory

Views30d =
CALCULATE(
    SUM(UsageMetrics[Views]),
    DATESINPERIOD(UsageMetrics[Date], TODAY(), -30, DAY)
)

UniqueUsers30d =
CALCULATE(
    SUM(UsageMetrics[UniqueUsers]),
    DATESINPERIOD(UsageMetrics[Date], TODAY(), -30, DAY)
)

PeakSlot =
VAR _peak =
    TOPN(
        1,
        SUMMARIZE(
            UsageMetrics,
            UsageMetrics[Date].[DayOfWeek],
            UsageMetrics[Hour],
            "v", SUM(UsageMetrics[Views])
        ),
        [v], DESC
    )
RETURN
    CONCATENATEX(_peak, FORMAT(UsageMetrics[Date], "ddd") & " " & UsageMetrics[Hour] & " AM")

LastRefreshRel =
VAR _last = MAX(RefreshHistory[StartedAt])
VAR _diff = NOW() - _last
RETURN
    SWITCH(TRUE(),
        _diff < 1/24,    INT(_diff*24*60) & " min ago",
        _diff < 1,       INT(_diff*24) & " hr "
                         & MOD(INT(_diff*24*60), 60) & " min ago",
        _diff < 7,       INT(_diff) & " d ago",
                         FORMAT(_last, "dd mmm")
    )

SLAState =
VAR _last  = MAX(RefreshHistory[StartedAt])
VAR _freq  = SELECTEDVALUE(DashboardDirectory[RefreshFrequency])
VAR _slaHr = SWITCH(_freq,
                "Real-time", 0.083,
                "Hourly",    2,
                "Daily",     26,
                "Weekly",    192, 999)
VAR _ok    = (NOW() - _last) * 24 <= _slaHr
RETURN IF(_ok, "● Within SLA", "● Behind SLA")

Refresh history 11

Last 10 runs only. Powered by the RefreshHistory table.

Edit interactions 13

A drillthrough page is read-only. The visuals must not cross-filter each other — the only filter is the page-level drillthrough.

  1. Select the Title card → ribbon → Format → Edit interactions
  2. Click each other visual on the page → set its interaction icon to None (no funnel, no highlight)
  3. Repeat for the Meta cards · Description card · Usage chart · KPI cards · Related matrix · History table
  4. Exit Edit interactions mode
Critical If you skip this step, clicking the area chart filters the description card to nothing because the chart's selected date doesn't match any row in DashboardDirectory. Test by clicking each visual after the build — every other visual must remain populated.

Test & publish 14

Run the drillthrough QA before pushing the .pbix to the workspace.

14.1 Drillthrough QA · 5 dashboards

Pick five varied rows from the directory (different categories, refresh frequencies, owners). For each:

  1. Right-click the dashboard name in the matrix → Drill through → Dashboard Detail
  2. Confirm: Title shows correct name. Category pill shows correct category. All six meta cells populate.
  3. Confirm: Description / Hero / Compact (whichever variant) shows the right content for that row.
  4. Confirm: Usage chart shows that dashboard's 90-day series, not the whole-portfolio sum.
  5. Confirm: Related grid lists peers in the same category (and excludes the current dashboard).
  6. Confirm: History table lists that dashboard's last 10 runs.
  7. Confirm: Status pill colour matches the SLA state correctly.
  8. Click Open dashboard ↗ — opens that specific dashboard's PBI URL.
  9. Click the Back button — returns to the landing page with original filters preserved.

14.2 Visual QA · 1280 × 900

  • No horizontal scroll. No vertical scroll if you chose the canvas-1100 approach.
  • Title gradient runs left-to-right with the deeper burgundy on the left.
  • All meta values render — none show "Blank" or empty.
  • Status pill colour matches the rule.
  • Only one left-column variant visible.
  • Related grid has at most 3 cards; never 4+.

14.3 Publish

  1. Home → Publish → choose the workspace
  2. In the Power BI service, open the report → confirm drillthrough still works in the browser (some interactions render differently online)
  3. If you embed via app, regenerate the app after publishing

Appendix C · Troubleshooting

SymptomFix
Title card shows "Blank" instead of dashboard name.The drillthrough binding didn't apply. Confirm DashboardName is in the page-level Drill through pane and you arrived via right-click → Drill through.
Title card shows ALL dashboard names concatenated.You used the field directly without SELECTEDVALUE aggregation, or "Keep all filters" is On — turn it Off.
Open dashboard ↗ opens the same URL every time.The Action → Web URL is hardcoded. Switch it to fx → SELECTEDVALUE(DashboardDirectory[URL]).
Status pill stays green even when refresh failed.The SLAState measure looks at the most recent run's timestamp, not status. Add a check: VAR _lastStatus = LOOKUPVALUE(RefreshHistory[Status], RefreshHistory[StartedAt], _last) and force "Behind SLA" when status ≠ "Success".
Usage chart shows whole-portfolio totals, not the selected dashboard.The visual is missing the drillthrough filter — drag DashboardName into the Visual-level filter, set "Filter type" to Auto, value = first.
Related matrix shows the current dashboard in the list.Add a Visual filter: DashboardName <> SELECTEDVALUE(DashboardDirectory[DashboardName]).
History table shows more than 10 rows.Visual-level Top N filter wasn't applied. Set N=10 by StartedAt descending.
Clicking the area chart blanks the description.You forgot Edit Interactions (Step 13). Switch chart's interaction with every other visual to None.
Compact variant chip text overflows.Either shorten the chip text (≤6 words) or widen each chip to 200 px and reduce the row to 2 chips.
Hero image is squashed.Image visual → Scaling = Fill, not Fit. Image must be ≥1280 px wide.
Title block gradient looks muddy / flat.The three-rectangle approximation needs the wash transparencies at exactly 35 %. Lower transparency darkens the centre; higher washes out the edges.
